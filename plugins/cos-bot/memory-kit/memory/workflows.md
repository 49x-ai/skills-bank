# Workflows

Reusable ways of working the assistant should follow.

## Update memory

When the user says "update memory":

1. Search `memory/` for relevant files.
2. Read `memory/MEMORY.md` and `memory/active.md`.
3. Identify only durable facts, decisions, constraints, or learnings.
4. Write a short update to the most relevant file.
5. Avoid duplicating the same fact in multiple places.

## Start project work

Before working on a task:

1. Read `PROTOCOL.md` and `memory/MEMORY.md`.
2. Read `memory/active.md`.
3. Search `memory/projects/` for the project name.
4. Check `memory/decisions.md` for relevant prior choices.

## End project work

At the end of meaningful work:

1. Summarize what changed.
2. Capture durable decisions in `decisions.md`.
3. Update open loops in `active.md`.
4. Don't save noisy details.

## Compact memory inbox

When asked to compact memory:

1. Read the current monthly `inbox/` file.
2. Move durable items to the right file.
3. Delete or archive temporary items.
4. Update `active.md` if priorities or open loops changed.
5. Keep the final memory short.
